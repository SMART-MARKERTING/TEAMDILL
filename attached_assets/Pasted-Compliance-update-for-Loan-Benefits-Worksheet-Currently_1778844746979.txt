Compliance update for Loan Benefits Worksheet. Currently the 
tool displays interest rates without APR, which violates 
TILA / Reg Z 12 CFR 1026.24. Add APR input fields, display 
APR alongside every rate at equal prominence, update compliance 
footer, and ensure ECOA / Equal Housing Opportunity language 
and logo are correct.

============================================
PART 1: ADD APR INPUT FIELDS
============================================

In the input panel (and the equivalent fields in private 
mode), add APR fields immediately adjacent to existing rate 
fields:

1. After "Existing Mortgage Rate" field:
   - Label: "Existing Mortgage APR (optional)"
   - Type: number, step 0.01, placeholder "6.987"
   - Variable name: existAPR

2. After "New Loan Rate" field:
   - Label: "New Loan APR" (REQUIRED — mark with asterisk)
   - Type: number, step 0.01, placeholder "6.752"  
   - Variable name: loanAPR
   - Required validation: cannot generate PDF or email without 
     this field populated. Show inline error: "APR is required 
     by federal law (TILA Reg Z) when displaying an interest 
     rate."

Existing rate fields stay where they are. APR sits immediately 
to the right (desktop) or directly below (mobile) at the same 
visual weight.

============================================
PART 2: DISPLAY APR ON THE WORKSHEET
============================================

Anywhere a rate is currently displayed, show APR at EQUAL 
prominence (same font size, same weight, same color). Side 
by side preferred. Stacked acceptable on mobile.

CHANGES TO MAKE:

A. "Your Current Picture" / Snapshot table row:
   Currently: "Existing Mortgage Balance | $320,000 @ 6.875%"
   Change to: "Existing Mortgage Balance | $320,000 @ 6.875% 
   Rate / 6.987% APR"
   If existAPR field is blank, show: "$320,000 @ 6.875% Rate 
   (APR not disclosed)"

B. "Your Current Picture" / New Loan row:
   Currently: "New Loan Amount | $380,000 @ 6.500%"
   Change to: "New Loan Amount | $380,000 @ 6.500% Rate / 
   6.752% APR"

C. "Your Three Paths Compared" table — the "Effective Rate on 
   New Loan" row:
   Currently: "6.500%"
   Change to: "6.500% Rate / 6.752% APR" in the Standard 
   Refinance column, and "4.438% effective" in the Accelerated 
   column with a small footnote indicator linking to the 
   compliance footer (since effective rate is not APR).

D. The banner callout text:
   Currently: "By redirecting $400/month back onto your new 
   loan's principal, your 6.500% loan effectively costs you 
   4.438%"
   Change to: "By redirecting $400/month back onto your new 
   loan's principal, your 6.500% Rate / 6.752% APR loan 
   effectively costs you 4.438% (illustrated effective rate, 
   not APR)"

E. The "How This Strategy Works" / explainer paragraphs:
   Wherever "6.500%" appears, change to "6.500% Rate / 6.752% 
   APR" on first mention, then "the new loan" on subsequent 
   mentions.

F. Key Numbers at a Glance table:
   Add a new row: "Stated interest rate" and "APR" both 
   shown explicitly.

============================================
PART 3: STYLING — EQUAL PROMINENCE RULE
============================================

CSS rules for any element displaying rate + APR:

.rate-display {
  display: inline-flex;
  align-items: baseline;
  gap: 4px;
}

.rate-display .rate-value,
.rate-display .apr-value {
  font-size: inherit;
  font-weight: inherit;
  color: inherit;
}

.rate-display .rate-label,
.rate-display .apr-label {
  font-size: 0.85em;
  opacity: 0.85;
}

Critical: do NOT make APR smaller, lighter, or grayer than 
the interest rate. They must be equal. The CSS above keeps 
the actual numbers at identical weight, only the "Rate" and 
"APR" word labels are slightly smaller (which is fine because 
they're labels, not the rate values themselves).
